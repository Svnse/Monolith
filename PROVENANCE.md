# Provenance

This document records the publication basis and source boundaries for the
Monolith `v1.0.0` public source release. It is an engineering record, not a
legal opinion.

## First-party source

The release tree was prepared from maintainer-controlled Monolith development
source in a dedicated curation checkout. The public branch begins with a new
root commit so machine-local state and private development history are not
ancestors of the published release. That history boundary does not
change the source's origin or ownership basis.

The intended first-party licensor is:

```text
Eryndel (Erick Ascano-Marin)
Copyright 2026
```

Selected first-party source and documentation are distributed under the root
[MIT License](LICENSE). That statement applies only where the maintainer owns or
has the right to license the material.

## AI-assisted development

AI systems were used as development tools under maintainer direction during
the private development process. The parentless public release does not
preserve every intermediary tool-authored commit. The maintainer is responsible
for reviewing and selecting material for publication; an AI author label is not
itself a license or proof of independent origin.

Contributors must identify copied/adapted material and preserve outside
attribution. “Generated by AI” is not an acceptable substitute for source and
rights review.

## Documentation and prompt material

The repository contains retained design notes, research, audits, prompts, and
training/probe material beyond the public documentation map. Inclusion in the
Git tree does not convert a plan into a shipped feature or establish provenance
by itself.

Before adding or promoting prompt/research corpora to the public surface:

1. identify the author and source;
2. distinguish original wording from quotation, adaptation, or private source;
3. confirm permission and quotation scope;
4. preserve required attribution; and
5. exclude the file when origin or rights remain unclear.

## Contributor provenance rule

Every new non-trivial asset or imported/adapted body of code/text should record:

- repository path;
- author/creator;
- source URL or source commit where applicable;
- original license or permission;
- modifications made;
- required notices; and
- reviewer/date.

Unknown provenance is a reason to withhold a file from a release, even when the
underlying feature can be deferred until after release.
